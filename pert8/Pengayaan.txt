1. Apa tujuan utama adanya interface dalam pemrograman berorientasi objek, dan bagaimana interface membantu menciptakan kontrak perilaku?
Jawab: Tujuan utama interface adalah untuk mendefinisikan kontrak perilaku (behavioral contract) yang harus dipenuhi oleh class yang mengimplementasikannya. Interface berperan sebagai spesifikasi atau blueprint yang mendeklarasikan "apa yang harus dilakukan" tanpa menentukan "bagaimana melakukannya". Interface membantu menciptakan kontrak perilaku dengan memaksa setiap class yang implements interface tersebut untuk menyediakan implementasi konkret dari semua method yang dideklarasikan. Dengan demikian, interface menjamin bahwa semua class yang mengimplementasikannya memiliki kemampuan (capability) yang sama, meskipun cara implementasinya bisa berbeda. Ini menciptakan konsistensi API dan memudahkan polymorphism, di mana kita bisa memperlakukan berbagai objek yang berbeda dengan cara yang seragam selama mereka mengimplementasikan interface yang sama.

2. Mengapa interface dapat memiliki banyak implementasi, dan apa makna konseptual dari kemampuan tersebut dalam desain sistem?
Jawab: Interface dapat memiliki banyak implementasi karena interface hanya mendefinisikan kontrak (method signature) tanpa implementasi konkret. Setiap class yang implements interface tersebut bebas memberikan implementasi yang sesuai dengan kebutuhan dan karakteristiknya masing-masing. Makna konseptualnya adalah "separation of concerns" dan "programming to an interface, not an implementation". Dalam desain sistem, ini berarti kita bisa memiliki berbagai class dengan implementasi yang sangat berbeda namun tetap memenuhi kontrak yang sama. Contoh: interface PaymentMethod bisa diimplementasikan oleh CreditCard, EWallet, BankTransfer, dan Cryptocurrency, masing-masing dengan logic pembayaran yang berbeda tetapi semua menyediakan method processPayment(). Ini memberikan fleksibilitas ekstrem untuk menambah implementasi baru tanpa mengubah code yang sudah ada (Open/Closed Principle).

3. Apa perbedaan mendasar antara extends dan implements dalam konteks interface, jika dilihat secara konsep tanpa melihat contoh kode?
Jawab: Secara konseptual, "extends" merepresentasikan hubungan IS-A (inheritance relationship) yang menyatakan bahwa suatu entity adalah spesialisasi atau turunan dari entity lain, mewarisi semua behavior dan state dari parent-nya. Sedangkan "implements" merepresentasikan hubungan CAN-DO atau HAS-CAPABILITY yang menyatakan bahwa suatu entity memiliki kemampuan atau dapat melakukan sesuatu yang didefinisikan oleh interface, tanpa harus mewarisi state atau behavior konkret. "Extends" fokus pada "what it is" (identitas dan hierarki), sementara "implements" fokus pada "what it can do" (capability dan role). Dengan extends, child class mewarisi implementasi dari parent class. Dengan implements, class harus menyediakan implementasi sendiri untuk semua method abstract di interface. Extends menciptakan tight coupling dan single inheritance tree, implements menciptakan loose coupling dan memungkinkan multiple capabilities.

4. Apa alasan diperkenalkannya default method dalam interface, dan bagaimana fitur ini memengaruhi keberlangsungan (compatibility) suatu sistem yang sudah berjalan?
Jawab: Default method diperkenalkan di Java 8 untuk mengatasi masalah backward compatibility dan evolution of interface. Sebelum Java 8, jika kita menambahkan method baru ke interface yang sudah digunakan oleh banyak class, semua class tersebut harus diupdate untuk mengimplementasikan method baru, atau akan terjadi compile error. Ini sangat merepotkan terutama untuk library yang sudah banyak digunakan. Dengan default method, kita bisa menambahkan method baru ke interface dengan implementasi default, sehingga class-class yang sudah ada tidak perlu diubah - mereka otomatis mewarisi implementasi default tersebut. Class yang memerlukan behavior khusus masih bisa meng-override default method. Fitur ini sangat memengaruhi keberlangsungan sistem karena memungkinkan interface untuk berkembang (evolution) tanpa breaking existing code, sehingga library dan framework bisa ditingkatkan secara bertahap tanpa memaksa semua user untuk melakukan perubahan besar-besaran pada code mereka.

5. Bagaimana konsep multiple interface memperluas fleksibilitas desain tanpa menimbulkan masalah kompleksitas yang biasa muncul pada multiple inheritance?
Jawab: Multiple interface memperluas fleksibilitas desain dengan memungkinkan class untuk memiliki multiple capabilities atau role tanpa masalah diamond problem yang muncul pada multiple inheritance. Masalah kompleksitas pada multiple inheritance muncul ketika ada ambiguitas tentang implementasi mana yang harus digunakan ketika dua parent class memiliki method dengan nama sama. Interface menghindari masalah ini karena interface hanya berisi deklarasi method tanpa implementasi (kecuali default method), sehingga tidak ada ambiguitas - class yang implements multiple interface harus menyediakan implementasi konkret sendiri untuk semua method. Bahkan jika ada konflik pada default method, compiler akan memaksa class untuk meng-override method tersebut dan menentukan implementasi yang jelas. Multiple interface memberikan fleksibilitas untuk membuat class yang memiliki banyak peran (role-based design) seperti ProgrammerMagang yang bisa KaryawanKontrak sekaligus AksesSistem, tanpa harus mewarisi implementasi konkret yang mungkin saling bertentangan. Ini mendukung composition over inheritance dan menghasilkan design yang lebih modular dan maintainable.
